# Session Handoff スキル

セッション終了時に、次回セッションへの引き継ぎ情報を自動生成するスキルです。

## 背景：なぜこのスキルが必要か

マルチセッションのタスク（例：ブログ記事の作成→レビュー→公開）で、セッション間のコンテキストが失われる問題が発生しました。手動でSESSION_HANDOFF.mdを作成して対処していましたが、フォーマットが不統一で、重要な情報が漏れることがありました。

## ワークフロー

### ステップ1: 現在の状態を収集

以下の情報を自動収集する：

1. **git status** — 変更済み・未追跡ファイルの一覧
2. **git diff --stat** — 変更の概要
3. **git log -5 --oneline** — 直近のコミット
4. **現在のブランチ** — 作業ブランチ名
5. **会話の要約** — このセッションで行った作業の要約

### ステップ2: ユーザーに確認

以下をユーザーに質問する：

- 残っているタスクはありますか？
- 次回セッションで最初にやるべきことは？
- 未コミットの変更をWIPコミットしますか？

### ステップ3: SESSION_HANDOFF.md を生成

プロジェクトルートに `SESSION_HANDOFF.md` を作成（既存がある場合は上書き確認）。

### テンプレート

```markdown
# Session Handoff

> 最終更新: YYYY-MM-DD HH:MM
> ブランチ: [branch-name]

## 今回のセッションで完了したこと

- [完了タスク1]
- [完了タスク2]

## 現在の状態

### 変更済みファイル
```
[git status の出力]
```

### 直近のコミット
```
[git log -5 --oneline の出力]
```

## 残タスク

- [ ] [残タスク1]
- [ ] [残タスク2]
- [ ] [残タスク3]

## 次回セッションで最初にやること

1. [最優先タスク]
2. [次のタスク]

## 重要な注意事項

- [コンテキスト情報（例：ユーザーのフィードバック待ち等）]
- [依存関係や前提条件]

## 参照すべきファイル

- [関連ファイルパス1] — [説明]
- [関連ファイルパス2] — [説明]
```

### ステップ4: 確認と保存

1. 生成した内容をユーザーに表示
2. 修正があれば反映
3. ファイルを保存
4. WIPコミットが必要な場合は実行

## 出力例

```markdown
# Session Handoff

> 最終更新: 2026-03-08 15:30
> ブランチ: main

## 今回のセッションで完了したこと

- セキュリティ総合ガイド記事のドラフト作成（656行）
- JSON-LD構造化データの実装（Article + FAQPage + BreadcrumbList）
- インラインCSSのレスポンシブ対応

## 現在の状態

### 変更済みファイル
- blog/security-comprehensive-guide.html (新規)
- blog/PLAN-security-comprehensive.md (新規)

## 残タスク

- [ ] ユーザーのフィードバックを記事に反映
- [ ] OGP画像生成: `python scripts/generate_ogp.py`
- [ ] blog-data.js に登録
- [ ] sitemap.xml に追加
- [ ] llms.txt に追加
- [ ] generate_ogp.py のARTICLES配列に追加

## 次回セッションで最初にやること

1. ユーザーからの記事フィードバックを確認
2. フィードバック反映後、blog-publisher スキルで公開登録

## 重要な注意事項

- 記事のトーンについてユーザー確認待ち
- FAQセクションは5件中4件完成、残り1件はデータ確認後に追加
```

## エラーハンドリング

| 状況 | 対処 |
|------|------|
| git が使えない環境 | git情報を省略し、ファイル一覧のみで生成 |
| 既存のSESSION_HANDOFF.md | 上書きするか、タイムスタンプ付きでアーカイブするか確認 |
| 未コミットの変更が大量 | 変更の要約を表示し、WIPコミットを強く推奨 |

## 注意事項

- SESSION_HANDOFF.md はgitにコミットしてよい（機密情報を含まないため）
- 次回セッション開始時にこのファイルを読み込むことで、コンテキストを復元できる
- このスキルは **作業の中断** に特化。作業の **開始** には使わない
