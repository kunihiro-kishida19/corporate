# ブログ記事品質チェックリスト

## セルフレビュー用（執筆完了後に実施）

### 論理チェック
- [ ] 統計データが主張を **直接** 支えているか
- [ ] 相関を因果として書いていないか
- [ ] 1つの事例で複数の提言を証明していないか
- [ ] 根拠が弱い箇所に「〜とされている」等の留保があるか

### 冗長チェック
- [ ] 同じメッセージが複数箇所に分散していないか
- [ ] 「まとめ」セクションが本文の単純な繰り返しでないか
- [ ] 不要な前置き（「言うまでもなく」等）がないか

### 出典チェック
- [ ] すべてのデータに出典（調査名・発行元・年）があるか
- [ ] 参考文献セクションに12〜16件のリンクがあるか
- [ ] 推測・伝聞が事実と区別されているか

### トーンチェック
- [ ] 「教える」ではなく「一緒に考える」スタンスか
- [ ] 上から目線の断定表現がないか
- [ ] 恐怖訴求（「危険です」「手遅れ」）がないか
- [ ] CxO読者の過去の判断を否定していないか
- [ ] 専門用語に補足説明があるか

### 構造チェック
- [ ] H1タイトル（40〜60文字、SEOキーワード含む）
- [ ] サマリボックス（4〜5箇条書き）
- [ ] H2 × 5〜8本、各H2下にH3 × 2〜3本
- [ ] 参考文献セクション（12〜16件）
- [ ] FAQセクション（4〜5件、スキーマ対応）
- [ ] CTAボックス
- [ ] 「記事一覧に戻る」リンク

### メタデータチェック
- [ ] og:description が100文字以上
- [ ] JSON-LD に Article + FAQPage + BreadcrumbList
- [ ] canonical URL が正しい絶対パス
- [ ] 日付フォーマットが正しい（表示: YYYY.MM.DD / 属性: YYYY-MM-DD）
