# 04. セキュリティ要件

> **ソース**: SITE_RULES.md §5

---

## 1. CSP（Content Security Policy）方針

- `default-src 'self'` — 自サイトのみ許可をベースとする
- `'unsafe-inline'` — スタイルとスクリプトに許可（GA4インラインスクリプトのため）
- 外部ドメインは **ホワイトリスト方式** で最小限に許可

### 1.1 許可ドメイン一覧

| ドメイン | 用途 | CSPディレクティブ |
|---------|------|-----------------|
| `fonts.googleapis.com` | Google Fonts CSS | `style-src` |
| `fonts.gstatic.com` | Google Fonts ファイル | `font-src` |
| `www.googletagmanager.com` | GA4 | `script-src`, `connect-src` |
| `www.google-analytics.com` | GA4 | `connect-src` |
| `*.google-analytics.com` | GA4 | `connect-src` |
| `formspree.io` | フォーム送信 | `connect-src` |
| `header-check.balcony-workers.workers.dev` | ヘッダーチェックAPI（セキュリティLP限定） | `connect-src` |

---

## 2. 外部リソース制限

- 新たな外部ドメインの追加は必ず CSP にも反映する
- CDN経由のライブラリ導入は避け、可能な限り self-hosted にする
- iframe の埋め込みは原則禁止

---

## 3. フォームセキュリティ

| 項目 | 設定 |
|------|------|
| プロバイダ | Formspree |
| Form ID | `xwvnnqjj`（公開可、クライアントサイド設計） |
| ハニーポット | `website` フィールド（hidden、botが入力すると送信拒否） |
| レート制限 | 30秒間隔 |
| エラー時フォールバック | `sales@balcony.co.jp` をユーザーに表示 |

---

## 4. クレデンシャル管理

- API キー、パスワード、トークンをリポジトリに格納しない
- Formspree ID は公開可（クライアントサイドで使用する設計）
- GA4 ID は公開可
- `.env`, `credentials.json`, `secrets.toml` 等は `.gitignore` に含める
