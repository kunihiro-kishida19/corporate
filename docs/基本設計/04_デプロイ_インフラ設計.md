# 04. デプロイ・インフラ設計

> **ソース**: 新規（GitHub Pages, CNAME, .nojekyll 等の実構成から記述）

---

## 1. ホスティング

| 項目 | 設定 |
|------|------|
| プラットフォーム | GitHub Pages |
| デプロイブランチ | `main` |
| デプロイ方式 | `main` への push で自動デプロイ |
| カスタムドメイン | `balcony.co.jp`（CNAME ファイルで設定） |

---

## 2. ドメイン設定

### 2.1 CNAME ファイル

リポジトリルートに `CNAME` ファイルを配置:

```
balcony.co.jp
```

### 2.2 DNS設定

GitHub Pages のカスタムドメイン設定に従い、DNS側で A レコードまたは CNAME レコードを設定。

---

## 3. GitHub Pages 固有設定

### 3.1 .nojekyll

リポジトリルートに `.nojekyll` ファイル（空ファイル）を配置。Jekyll のビルドプロセスをスキップし、静的ファイルをそのまま配信する。

### 3.2 ディレクトリ構成

GitHub Pages はリポジトリのルートをそのまま公開する。`/service/index.html` は `https://balcony.co.jp/service/` でアクセス可能。

---

## 4. ファイル配信

| ファイル | パス | 用途 |
|---------|------|------|
| `CNAME` | `/CNAME` | カスタムドメイン設定 |
| `.nojekyll` | `/.nojekyll` | Jekyll ビルドスキップ |
| `robots.txt` | `/robots.txt` | クローラー制御 |
| `sitemap.xml` | `/sitemap.xml` | 検索エンジン用サイトマップ |
| `llms.txt` | `/llms.txt` | AI検索エンジン用情報 |
| `favicon.svg` | `/favicon.svg` | ファビコン |

---

## 5. HTTPS

GitHub Pages はカスタムドメインに対して自動で HTTPS (Let's Encrypt) を提供。全 URL は `https://` で配信される。

---

## 6. キャッシュ

GitHub Pages のデフォルトキャッシュポリシーに準拠。カスタムの Cache-Control ヘッダーは設定不可（GitHub Pages の制約）。
