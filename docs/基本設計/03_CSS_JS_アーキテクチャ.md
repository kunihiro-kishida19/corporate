# 03. CSS・JS アーキテクチャ

> **ソース**: SITE_RULES.md §3.1, §4.1, §4.5

---

## 1. ファイル構成

### 1.1 共通ファイル

| ファイル | 内容 |
|---------|------|
| `/css/style.css` | 全ページ共通スタイル（CSS変数、リセット、ナビ、フッター、ヒーロー、アニメーション） |
| `/js/main.js` | ナビ制御、スクロールスパイ、フローティングCTA、フォーム送信、revealアニメーション |
| `/js/blog-data.js` | ブログ記事データ（`BLOG_DATA` 配列） |
| `/js/news-data.js` | ニュースデータ（`NEWS_DATA` 配列） |

### 1.2 セキュリティLP 固有ファイル

セキュリティLPは共通 CSS/JS を **使用しない**。独自のデザインシステムを持つ。

| ファイル | 内容 |
|---------|------|
| `/security/style.css` | ダークテーマ全体（独立したCSS変数） |
| `/security/main.js` | 粒子アニメーション、スクロールスパイ、FAQ、フォーム送信 |

### 1.3 ツール固有ファイル

| ファイル | 内容 |
|---------|------|
| `/tools/header-check/index.html` | ヘッダーチェックツール（HTML + 埋め込みCSS/JS） |

## 2. CSS設計

### 2.1 CSS変数

全てのカラー・スペーシング・トランジションはCSS変数で管理。詳細は [サイト要件/05_用語集_ブランドガイド.md](../サイト要件/05_用語集_ブランドガイド.md) を参照。

### 2.2 レスポンシブ ブレークポイント

| ブレークポイント | 用途 |
|----------------|------|
| 1200px | デスクトップ（ワイド） |
| 1024px | デスクトップ（標準） |
| 768px | タブレット |
| 640px | モバイル |
| 480px | 小型モバイル |

### 2.3 アニメーション

- **スクロールアニメーション**: IntersectionObserver + `.reveal` クラス
- **ヒーロー**: スタガーアニメーション（`animation-delay` で順次表示）
- **ナビ**: `backdrop-filter: blur()` + スクロール時 compact 化
- **セキュリティLP**: Canvas 粒子アニメーション（160個の星）

---

## 3. JS設計

### 3.1 main.js の責務

1. **ナビゲーション制御**: スクロール時の compact 化、モバイルハンバーガーメニュー
2. **スクロールスパイ**: アクティブページのハイライト
3. **フローティングCTA**: Hero 通過後に表示
4. **フォーム送信**: Formspree への fetch POST
5. **スクロールアニメーション**: `.reveal` の IntersectionObserver 監視

### 3.2 データファイル

- `blog-data.js`: `BLOG_DATA` 配列 — ブログ一覧の描画に使用
- `news-data.js`: `NEWS_DATA` 配列 — ニュース一覧の描画に使用

---

## 4. 独立性の原則

- 共通CSSの変更はセキュリティLPに影響しない（独立したCSS）
- セキュリティLP固有の変更は他ページに影響しない
- フォーム送信は独自実装だが、Formspree ID は共通（`xwvnnqjj`）
